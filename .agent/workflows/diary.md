# 更新開發日記工作流程

## 步驟

1. 讀取 `/Users/lawliet/OVEREND/DOCS/` 資料夾中所有 Markdown 文件

2. 檢視當前開發日記 `/Users/lawliet/OVEREND/DOCS/DEVELOPMENT_DIARY.md`

3. 從最近的對話記錄和文件變更中提取重要開發資訊：
   - 新增或修改的功能
   - 解決的問題
   - 技術決策
   - 已完成的任務

4. 更新 `DEVELOPMENT_DIARY.md` 的開發時間線區段，新增今日的開發紀錄

5. 更新以下區段（如有需要）：
   - 核心功能開發進度
   - 問題解決紀錄
   - 待開發功能
   - 編譯狀態

6. 同步更新 `PROJECT_STATUS.md` 中的：
   - 更新日期
   - 專案進度百分比
   - 已完成/進行中功能清單

7. 確認更新內容並請用戶審閱
